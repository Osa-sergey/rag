## 1. Foundations (Основы)

### 1.1 Доступность (Accessibility)

Доступность — базовый принцип M3, интегрированный *во все компоненты с самого начала*.

#### Ключевые требования

| Параметр | Минимальное значение | Контекст |
|:---|:---|:---|
| Контрастность текста (мелкий) | **4.5 : 1** | Основной текст, подписи |
| Контрастность текста (крупный) | **3 : 1** | Заголовки, иконки, границы |
| Размер touch-target | **48 × 48 dp** | Сенсорные экраны |
| Размер pointer-target | **44 × 44 dp** | Мышь / трекпад |
| Масштабирование текста | до **200 %** | Пользовательские настройки |

#### UX-советы

- **Фокус**: Все интерактивные элементы должны иметь видимый индикатор фокуса (outline / ring).
- **Tab-order**: Логичная последовательность навигации клавиатурой; семантическая HTML-разметка.
- **ARIA**: Экраны для скринридеров — используйте `role`, `aria-label`, `aria-live` при необходимости.
- **Цвет — не единственный индикатор**: Ошибки, состояния, статусы дублируйте иконками и текстом.

---

### 1.2 Раскладка (Layout)

#### Window Size Classes

M3 определяет три опорных класса размера окна:

| Класс | Ширина | Типичные устройства |
|:---|:---|:---|
| **Compact** | < 600 dp | Телефоны (портрет) |
| **Medium** | 600 – 840 dp | Планшеты (портрет), складные |
| **Expanded** | > 840 dp | Планшеты (ландшафт), десктоп |

#### Canonical Layouts (Каноничные раскладки)

Вместо произвольных гридов M3 рекомендует готовые паттерны:

- **List-Detail** — список слева, детали справа (почта, мессенджеры).
- **Feed** — вертикальная лента контента (новости, соцсети).
- **Supporting Pane** — основной контент + боковая панель с доп. информацией.

#### Система отступов

- **Базовая сетка**: 8 dp (для более плотных зон — 4 dp).
- Используйте `margin`, `padding`, `spacer` для управления плотностью информации.
- **Bidirectionality**: Полная поддержка RTL-языков — компоненты зеркалятся автоматически.

---

### 1.3 Дизайн взаимодействия (Interaction Design)

#### Состояния интерактивных элементов

Каждый интерактивный компонент проходит через набор состояний:

```
Enabled → Hovered → Focused → Pressed → Dragged → Disabled
```

> [!IMPORTANT]
> Визуальная индикация состояния обязательна. Пользователь должен моментально понимать, что элемент активен, отключён или находится в фокусе.

#### Жесты

| Жест | Действие |
|:---|:---|
| Tap | Основное действие |
| Long press | Контекстное действие / тултип (мобильные) |
| Swipe | Навигация, dismiss |
| Scroll | Прокрутка контента |
| Pinch | Масштабирование |
| Predictive Back | Превью возврата назад (Android) |

#### Обратная связь

- Реакция UI на ввод должна быть **мгновенной** (< 100 мс визуально).
- Используйте ripple-эффект, анимации состояний, haptic feedback.

---

### 1.4 Design Tokens

Design Token — именованная переменная, хранящая стилевое значение (цвет, шрифт, отступ, радиус).

```
Токен → Роль → Компонент
md.sys.color.primary → используется в Button → окрашивает фон кнопки
```

**Зачем**:
- Единый источник правды для design ↔ code.
- Смена темы (light / dark / brand) меняет только значения токенов.
- Гарантирует консистентность во всей системе.

## 2. Styles (Стили)

### 2.1 Цвет (Color System)

#### Цветовые роли

M3 использует семантические роли вместо жёстких HEX-значений:

| Роль | Назначение |
|:---|:---|
| **Primary** | Главный акцент: кнопки, переключатели, активные состояния |
| **Secondary** | Второстепенный акцент: менее выделенные элементы |
| **Tertiary** | Третичный акцент: баланс, дополнение к primary/secondary |
| **Surface** | Фоны и крупные области |
| **Surface Container** | Фон вложенных контейнеров (карточки, панели) |
| **On-{Role}** | Текст и иконки поверх роли (напр., `on-primary`) |
| **{Role}Container** | Заливка выделенных компонентов (кнопок, чипов) |
| **Outline** | Границы, разделители |
| **Error** | Состояния ошибок |

#### Dynamic Color

- Палитра автоматически генерируется из исходного цвета (обои, бренд).
- Гарантирует доступность контраста через **тональные палитры** (0–100).
- Если Dynamic Color недоступен — используется статическая baseline-схема.

#### On-{Role} правило

> [!TIP]
> Всегда используйте пару `role` / `on-role`. Например, текст на `primaryContainer` должен быть `onPrimaryContainer`. Это гарантирует контраст ≥ 3:1.

#### Surface Container иерархия

```
Surface < Surface Container Low < Surface Container < Surface Container High < Surface Container Highest
```

Чем выше элемент в иерархии, тем более выраженный фон он получает.

---

### 2.2 Типографика (Typography)

#### Type Scale (5 ролей × 3 размера = 15 стилей)

| Роль | Large | Medium | Small | Назначение |
|:---|:---|:---|:---|:---|
| **Display** | 57 / 64 | 45 / 52 | 36 / 44 | Героические заголовки, крупные метрики |
| **Headline** | 32 / 40 | 28 / 36 | 24 / 32 | Заголовки на малых экранах |
| **Title** | 22 / 28 | 16 / 24 | 14 / 20 | Заголовки карточек, диалогов, секций |
| **Body** | 16 / 24 | 14 / 20 | 12 / 16 | Основной текст, параграфы |
| **Label** | 14 / 20 | 12 / 16 | 11 / 16 | Кнопки, подписи, поля ввода |

> Формат: `size / line-height` (dp).

#### UX-советы по типографике

- **Шрифт по умолчанию**: Roboto. Рекомендуется Roboto Flex (variable font).
- **Line height**: ≈ 1.5× от размера шрифта для Body и Label.
- **Табулярные цифры**: Используйте monospaced figures для таймеров, таблиц, счётчиков — предотвращает layout shift.
- **Emphasized-стили**: Увеличенный weight для создания иерархии.

---

### 2.3 Форма (Shape)

#### Shape Scale (10 ступеней)

| Токен | Радиус | Типичные компоненты |
|:---|:---|:---|
| `None` | 0 dp | — |
| `Extra Small` | 4 dp | Чипы, малые элементы |
| `Small` | 8 dp | Текстовые поля, чекбоксы |
| `Medium` | 12 dp | Карточки, диалоги |
| `Large` | 16 dp | FAB, нижние панели |
| `Large Increased` | 20 dp | — |
| `Extra Large` | 28 dp | Крупные контейнеры |
| `Extra Large Increased` | 32 dp | — |
| `Extra Extra Large` | 48 dp | — |
| `Full` | 50 % | Полностью скруглённые (капсулы, круги) |

#### UX-советы

- **Ассиметрия**: Для сгруппированных элементов (меню, сплит-кнопки) внутренние углы могут иметь меньший радиус.
- **Семейство формы**: По умолчанию — `rounded`. Можно сменить на `cut` (прямые срезы) для бренда.
- **Shape morphing**: Анимируйте смену формы при переходах между состояниями.

---

### 2.4 Возвышение (Elevation)

M3 использует **два механизма** возвышения:

#### Тональное возвышение (Tonal Elevation) — основной метод

- Высшие элементы получают более выраженный тон (tint) от `primary`.
- Реализуется через Surface Container роли.
- **Отлично работает** и в светлой, и в тёмной темах.

#### Теневое возвышение (Shadow Elevation) — вспомогательный

- Классическая тень (drop shadow).
- Используется **избирательно**: FAB, модальные окна, элементы на сложном фоне.
- В тёмной теме тени менее эффективны.

| | Тональное | Теневое |
|:---|:---|:---|
| **Механизм** | Цветовой тон / Surface Roles | Drop Shadow |
| **Цель** | Иерархия и разделение | Акцент на «подъём» |
| **Эффективность в тёмной теме** | ✅ Отличная | ⚠️ Ограничена |
| **Рекомендация** | Предпочтительный метод | Используйте для дополнительного фокуса |

> [!TIP]
> Начинайте с тонального возвышения. Добавляйте тень только когда элемент должен визуально «оторваться» от фона.

### 2.5 Движение (Motion)

#### Современная модель: Physics-based Springs

M3 перешла от статичных `easing + duration` к **пружинной физике**:

| Параметр | Описание |
|:---|:---|
| **Stiffness** | Жёсткость пружины — скорость анимации |
| **Damping** | Затухание — количество «отскоков» |

#### Motion Schemes

| Схема | Характер | Когда использовать |
|:---|:---|:---|
| **Expressive** (по умолчанию) | Выразительная, с лёгким bounce | Hero-моменты, ключевые переходы |
| **Standard** | Утилитарная, без bounce | Плотные интерфейсы, утилиты |

#### Скорости (Spring Speeds)

| Скорость | Применение |
|:---|:---|
| **Default** | Большинство анимаций |
| **Fast** | Мелкие элементы (чипы, бейджи, тогглы) |
| **Slow** | Крупные элементы / полноэкранные переходы |

#### Legacy-система (для совместимости)

Некоторые платформы (Web, iOS) всё ещё используют easing-кривые:

| Transition | Easing Set |
|:---|:---|
| **Emphasized** | Основной для M3. Быстрый старт, мягкое замедление. |
| **Standard** | Утилитарный. Симметричный, быстрый. |

#### UX-советы по анимациям

- Анимация должна быть **информативной** (сообщать о причинно-следственной связи).
- Анимация должна быть **сфокусированной** (привлекать внимание к нужному месту).
- Анимация должна быть **экспрессивной** (отражать характер бренда).
- **Никогда** не блокируйте пользователя длинной анимацией.

## 3. Компоненты

### 3.1 Кнопки (Buttons)

#### Иерархия кнопок

| Тип | Акцент | Когда использовать |
|:---|:---|:---|
| **Filled** | 🔴 Высокий | Финальные действия: «Сохранить», «Подтвердить» |
| **Filled Tonal** | 🟠 Средне-высокий | Видимые, но не основные действия |
| **Elevated** | 🟡 Высокий | На сложном / узорчатом фоне |
| **Outlined** | 🟢 Средний | Альтернативные действия: «Отмена», «Назад» |
| **Text** | ⚪ Низкий | Минимальные действия: в карточках, диалогах |

#### Кнопки с иконками (Icon Buttons)

| Тип | Назначение |
|:---|:---|
| **Default** | Запуск действия / открытие меню |
| **Toggle** | Бинарное состояние (избранное on/off) |

> При toggle: unselected → **outlined** иконка; selected → **filled** иконка.

#### UX-правила для кнопок

> [!IMPORTANT]
> - Не размещайте несколько Filled-кнопок на одном экране — они конкурируют за внимание.
> - Используйте **sentence case** для надписей.
> - Текст кнопки — **в одну строку**, без переносов.
> - Минимальный touch-target: 48 × 48 dp.
> - Опциональные иконки (leading / trailing): стандартный размер 20 dp.
> - Ширина кнопки адаптивна, но не уже содержимого текста.

---

### 3.2 FAB (Floating Action Button)

#### Варианты FAB

| Тип | Размер | Применение |
|:---|:---|:---|
| **FAB** | Standard | Большинство экранов |
| **Medium FAB** | Чуть больше | Важные действия без лишних размеров |
| **Large FAB** | Крупный | Крупные экраны (планшеты, десктоп) |
| **Extended FAB** | Иконка + текст | Когда иконка одна неоднозначна |

> ⚠️ Small FAB более **не рекомендуется** в актуальных M3 guidelines.

#### UX-правила для FAB

- **Одно основное действие на экран**: «Создать», «Добавить», «Поделиться».
- **Один FAB на экране** — иначе конкуренция за внимание.
- FAB видна поверх всего контента и **не скрывается при скролле**.
- Иконка должна **чётко передавать действие**.
- Не дублируйте действие FAB другими кнопками на экране.
- **Цвет**: Используйте `primaryContainer` / `onPrimaryContainer` (тональные стили).
- «Surface» FAB более **не рекомендуется**.

#### Адаптивное размещение

| Экран | Позиция FAB |
|:---|:---|
| Compact (телефон) | Нижний правый угол |
| Expanded (десктоп) | Интеграция с Navigation Rail (верхний левый) |

### 3.3 Карточки (Cards)

#### Три варианта

| Тип | Визуал | Акцент |
|:---|:---|:---|
| **Elevated** | Drop shadow | Средний |
| **Filled** | Сплошной фон | Низкий |
| **Outlined** | Граница (stroke) | Высокий |

#### UX-правила для карточек

- Используйте карточки для **связанной информации по одной теме**.
- Карточки — отличные **точки входа** в более глубокий контент.
- **Не злоупотребляйте**: если whitespace, заголовки или разделители справляются — карточки не нужны.
- Элементы внутри карточки расположены по **визуальной иерархии**.
- Карточка может быть **actionable** (вся карточка — кнопка) или **non-actionable** (контейнер).

> [!CAUTION]
> Не размещайте множество actionable-элементов (кнопок) внутри уже actionable карточки — это создаёт путаницу с focus и touch-target.

- **Текст поверх изображений**: Используйте scrim (полупрозрачный слой) для читаемости.
- **Скролл внутри карточки**: Запрещён. Если контент не умещается — увеличивайте карточку или отправляйте на отдельный экран.
- **Адаптивность**: На мобильных — во всю ширину. На больших экранах — гриды, горизонтальные ряды, карусели.

### 3.4 Навигация

#### Навигационные компоненты и их контекст

| Компонент | Экран | Кол-во пунктов | Описание |
|:---|:---|:---|:---|
| **Navigation Bar** | Compact (телефон) | 3–5 | Фиксированная полоска внизу |
| **Navigation Rail** | Medium+ (планшет) | 3–7 | Вертикальная полоса по ведущему краю |
| **Navigation Drawer** | Expanded | 5+ | Полная боковая панель |

#### UX-правила

- **Navigation Bar**: Не используйте на десктопе. Фиксирован, не скроллится. Активный пункт выделяется pill-индикатором.
- **Navigation Rail**: Может содержать FAB наверху. Поддерживает дополнительный menu icon для раскрытия.
- **Navigation Drawer**: Для сложной иерархии; в M3 рекомендуется использовать **expanded navigation rail** вместо drawer.
- **Адаптивный переход**: При изменении размера окна компонент навигации автоматически переключается (Bar → Rail → Drawer).

### 3.5 Панели приложения (App Bars)

#### Top App Bar

| Вариант | Описание |
|:---|:---|
| **Small** | Компактный, для стандартных экранов |
| **Medium Flexible** | Расширенный заголовок, коллапсируется при скролле |
| **Large Flexible** | Крупный заголовок, коллапсируется при скролле |

#### UX-правила

- Отображает заголовок страницы, навигацию (back), primary actions.
- При скролле: фоновый fill-color отделяет панель от контента (не drop shadow).
- Может быть фиксированной или скрываемой.

#### Bottom App Bar

- Располагает навигацию и ключевые действия внизу.
- Часто содержит FAB (anchored или overlapping).

---

### 3.6 Диалоги (Dialogs)

#### Назначение

Диалоги — **блокирующие** компоненты для критически важной информации.

#### UX-правила

> [!WARNING]
> Используйте диалоги **экономно**. Частые диалоги раздражают и снижают конверсию.

- **Только для критичных решений**: удаление данных, подтверждение необратимых действий.
- Появляются поверх всего контента, требуют взаимодействия для закрытия.
- Заголовок и кнопки — краткие и ясные.
- **Фокус**: При открытии фокус перемещается на первый интерактивный элемент.
- Используйте scrim (затемнение фона) для подчёркивания блокировки.

---

### 3.7 Bottom Sheets

#### Назначение

Вторичное содержимое, быстрые действия, контекстные задачи — **менее disruptive** чем диалоги.

#### UX-правила

- Выезжает снизу, фиксируется к краю.
- Блокирует остальной UI (modal) пока не закрыт.
- Закрытие: свайп вниз, tap по scrim, или явное действие.
- **Начальная высота**: ≤ 50 % экрана. При необходимости — расширяется до full-screen.
- Хорош для сохранения контекста — пользователь остаётся на текущем экране.

---

### 3.8 Snackbar

#### Назначение

Неблокирующая **временная обратная связь** о фоновых процессах.

#### UX-правила

- Появляется внизу экрана.
- **Auto-dismiss** через 4–10 секунд или действие пользователя.
- Показывайте **только один** Snackbar одновременно.
- Не перекрывайте FAB или критичную навигацию.
- Опциональная кнопка действия (напр., «Отменить»).
- Для сложных сообщений — используйте другой компонент (Banner, Dialog).

---

### 3.9 Banner

#### Назначение

**Постоянные** сообщения, требующие внимания или действия.

#### UX-правила

- Размещается вверху экрана, под Top App Bar.
- Остаётся видимым до dismiss или разрешения причины (напр., восстановление связи).
- Более заметен чем Snackbar, менее навязчив чем Dialog.
- Идеален для: «Нет подключения», «Обновление доступно», «Требуется верификация».

---

### 3.10 Текстовые поля (Text Fields)

#### Два варианта

| Вариант | Визуал | Акцент | Лучше подходит |
|:---|:---|:---|:---|
| **Filled** | Цветной фон + нижняя линия | Высокий | Диалоги, короткие формы |
| **Outlined** | Полная рамка | Низкий | Длинные формы, плотные layouts |

> [!IMPORTANT]
> **Не смешивайте** Filled и Outlined поля в пределах одной формы / секции. Выберите один стиль.

#### Анатомия текстового поля

```
┌────────────────────────────────┐
│  🔍  Label (floating up)   ✕  │
│  Input text here...            │
│  Helper / Error text           │
└────────────────────────────────┘
```

- **Label**: Обязательна. Короткая, без усечения. При фокусе — «всплывает» вверх.
- **Helper text**: Подсказка до ошибки (напр., «Минимум 8 символов»).
- **Error text**: Заменяет helper при ошибке — краткое, конкретное.
- **Иконки**: Leading (тип ввода), trailing (действие: очистить, показать пароль).

#### UX-правила для валидации

- **Помощь важнее наказания**: Helper text предупреждает до ошибки.
- Ошибка дублируется иконкой + текстом (не только цветом).
- Минимальный контраст границы Outlined поля: 3:1 к фону.
- Активные состояния: `Enabled → Hovered → Focused → Error → Disabled` — каждое визуально отличимо.

---

### 3.11 Чипы (Chips)

#### Четыре варианта

| Тип | Назначение | Пример |
|:---|:---|:---|
| **Filter** | Фильтрация контента (замена toggle/checkbox) | Фильтр категорий товаров |
| **Input** | Введённые данные в компактной форме | Email-контакты, теги |
| **Suggestion** | Динамические подсказки | Quick replies в чате |
| **Assist** | Контекстные действия | «Добавить в календарь» |

#### UX-правила

- Чипы **всегда в группе** — не одиночные элементы.
- Контрастность текста ≥ 3:1.
- При переполнении: **Reflow** (развёртывание) или **Menu** (кнопка «ещё»). Горизонтальный скролл — **не рекомендуется**.
- Elevation по умолчанию: 0. Повышается только при размещении на изображении.
- Опциональные leading-иконки. Input chip может содержать аватар.
- Filter chip при выборе отображает checkmark.

---

### 3.12 Элементы выбора (Selection Controls)

#### Когда что использовать

| Контрол | Выбор | Поведение |
|:---|:---|:---|
| **Radio Button** | Один из набора | Выбор нового → предыдущий сбрасывается |
| **Checkbox** | Ноль или более | Независимые; поддерживает `indeterminate` |
| **Switch** | Вкл / Выкл | Эффект **мгновенный** |

#### UX-правила

> [!IMPORTANT]
> - **Switch**: Эффект применяется сразу. Не используйте Switch, если нужна отдельная кнопка «Сохранить».
> - **Checkbox**: Если выбор требует «Сохранить» — используйте checkbox, не switch.
> - Не смешивайте метафоры: если опции взаимоисключающие — radio button, а не checkbox.

- **Switch НЕ для противоположных опций** (напр., «Список» / «Карта»). Для этого — Segmented Button.
- Клик по label = клик по контролу (увеличивает hit area).
- Выбранное состояние визуально выделено (цвет + форма).

---

### 3.13 Списки (Lists)

#### Анатомия элемента списка

```
┌──────────────────────────────────────────┐
│ [Avatar]  Label Text         [Trailing]  │
│           Supporting text (1–3 lines)     │
└──────────────────────────────────────────┘
```

#### UX-правила

- Элементы — **вертикальный непрерывный индекс**.
- Label text: краткий. Supporting text: 1–3 строки макс.
- Leading / trailing элементы (иконки, аватары, чекбоксы) — **в одинаковой позиции** для каждого элемента.
- На больших экранах — широкие поля, multi-column или оборачивание в List-Detail layout.
- При больших данных — добавьте поиск и сортировку.

---

### 3.14 Разделители (Dividers)

| Тип | Назначение |
|:---|:---|
| **Full-width** | Разделение несвязанных секций |
| **Inset** | Разделение связанных элементов (выровнен по контенту) |
| **Без разделителя** | Whitespace / margins для повторяющихся списков |

#### UX-советы

- Используйте **экономно** — слишком много линий = визуальный шум.
- Если оба типа на одном экране — они должны усиливать иерархию.
- Для повторяющихся списков часто достаточно **whitespace** вместо divider.

---

### 3.15 Вкладки (Tabs)

#### Primary vs Secondary

| | Primary Tabs | Secondary Tabs |
|:---|:---|:---|
| **Уровень** | Top-level навигация | Sub-level внутри Primary |
| **Размещение** | Под Top App Bar | В теле контента |
| **Переключение** | Меняет основной view | Фильтрует внутри текущего view |

#### UX-правила

- **Не смешивайте** Primary и Secondary в одном tab bar.
- **Fixed**: Когда все вкладки помещаются.
- **Scrollable**: Когда вкладок больше ширины экрана.
- Labels — краткие и понятные. Опциональные иконки — только если значимые.

---

### 3.16 Сегментные кнопки (Segmented Buttons)

- Содержат **2–5 сегментов**.
- Labels — короткие; **все текст ИЛИ все иконки** (не смешивать).
- При выборе: цвет + checkmark (множественные индикаторы).
- Outline contrast ≥ 3:1.
- В M3 Expressive заменяются на **Connected Button Groups**.

---

### 3.17 Меню (Menus)

- Временный список вариантов / действий.
- Закрывается при выборе или tap вне меню.
- **Группировка**: разделители / отступы для логических блоков.
- **Фокус**: при открытии — на первом пункте.
- Touch target каждого пункта ≥ 48 dp.

---

### 3.18 Индикаторы прогресса (Progress Indicators)

| Вариант | Форма | Описание |
|:---|:---|:---|
| **Linear** | Полоса | Горизонтальный прогресс-бар |
| **Circular** | Кольцо | Компактный индикатор |

| Состояние | Когда | Визуал |
|:---|:---|:---|
| **Determinate** | Известен прогресс | Заполняется 0 → 100 % |
| **Indeterminate** | Неизвестна длительность | Непрерывная анимация |

#### UX-правила

- Один и тот же процесс → один и тот же тип индикатора во всём приложении.
- Linear determinate: используйте **stop indicator** для соответствия контрастным требованиям.

---

### 3.19 Слайдеры (Sliders)

| Вариант | Описание |
|:---|:---|
| **Standard** | Одно значение (громкость, яркость) |
| **Centered** | Диапазон с центром (положительные/отрицательные) |
| **Range** | Два значения (мин – макс) |

#### Анатомия

- **Track**: Полоса диапазона.
- **Thumb** (ручка): Основной элемент управления.
- **Value Label**: Опциональный попап с текущим значением.
- **Tick Marks**: Опциональные деления на шкале.

#### UX-правила

- Handle меняет форму/размер при взаимодействии (visual feedback).
- Полезен когда точное значение вторично, а важнее **относительное положение**.

---

### 3.20 Бейджи (Badges)

| Вариант | Содержимое | Пример |
|:---|:---|:---|
| **Small** (dot) | Нет текста | Индикатор нового контента |
| **Large** | Число / текст (≤ 4 символа) | Количество непрочитанных |

#### UX-правила

- Крепится к **верхнему trailing-краю** иконки.
- Контраст ≥ 3:1.
- Используйте дефолтные цветовые маппинги (`error` для красных бейджей).

---

### 3.21 Тултипы (Tooltips)

| Тип | Содержимое | Триггер |
|:---|:---|:---|
| **Plain** | Короткий текст-пояснение | Hover / long-press |
| **Rich** | Подзаголовок, кнопки, ссылки | Hover / long-press |

#### UX-правила

- **Plain**: Идеальны для icon-only кнопок — объясняют действие.
- **Rich**: Для более развёрнутых пояснений.
- ⚠️ **Не прячьте критичную информацию** в тултипы — пользователь может их не увидеть.
- Для обязательной информации — используйте Dialog или inline-текст.

---

### 3.22 Поиск (Search)

#### Анатомия

```
┌────────────────────────────────────────┐
│  🔍  Search text here...        [👤]  │
│  ─ Suggested result 1                 │
│  ─ Suggested result 2                 │
│  ─ Suggested result 3                 │
└────────────────────────────────────────┘
```

#### UX-правила

- Leading icon: линза. Trailing icon: очистка / аватар.
- При фокусе: может расширяться или переходить в full-screen.
- Результаты появляются в списке под полем по мере ввода.
- Чёткий контраст и доступные labels для всех иконок.

## 4. Матрица выбора компонента

### Когда что использовать для обратной связи

| Приоритет | Блокирующий? | Компонент | Пример |
|:---|:---|:---|:---|
| 🔴 Критичный | Да | **Dialog** | «Удалить аккаунт?» |
| 🟠 Высокий | Да (modal) | **Bottom Sheet** | Выбор действия по элементу |
| 🟡 Средний | Нет (persistent) | **Banner** | «Нет подключения к сети» |
| 🟢 Низкий | Нет | **Snackbar** | «Письмо помечено как прочитанное» |

### Когда что использовать для выбора

| Сценарий | Компонент |
|:---|:---|
| Один из нескольких (≤ 5) | **Radio Button** |
| Множественный выбор | **Checkbox** |
| Вкл/Выкл (мгновенный эффект) | **Switch** |
| 2–5 опций inline | **Segmented Button** |
| Фильтрация набора | **Filter Chip** |
| Значение из диапазона | **Slider** |

### Когда что использовать для навигации

| Экран | Пунктов | Компонент |
|:---|:---|:---|
| Compact | 3–5 | **Navigation Bar** |
| Medium | 3–7 | **Navigation Rail** |
| Expanded | 5+ | **Navigation Drawer** / **Expanded Rail** |
| Sub-navigation | 2–8 | **Tabs** (Primary / Secondary) |
